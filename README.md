# Задание 1
1)  Высокая нагрузка на базу данных при чтении популярных записей
Например, новостной сайт. Статья набирает миллионы просмотров в первый час после публикации. Без кеширования каждый просмотр вызывал бы SQL‑запрос к БД, что быстро приведёт к её перегрузке. Кеширование содержимого статьи (например, в Redis или Memcached) позволяет отдавать её прямо из памяти, минуя базу данных.

2) Долгие вычисления или сложные агрегации
В аналитической системе отчёт строится на основе миллионов записей и занимает несколько секунд. Если отчёт запрашивают часто (например, каждые 5 минут), можно кешировать результат на 5 минут. Тогда пользователи мгновенно получают готовый отчёт, а тяжёлые вычисления выполняются реже.
